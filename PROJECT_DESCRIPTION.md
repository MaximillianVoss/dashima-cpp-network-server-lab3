# Описание проекта

## Назначение

Проект выполнен для лабораторной работы 3 «Иерархия классов», вариант 20 «Сетевой сервер». Приложение моделирует работу сетевого сервера с таблицей передач. Таблица хранит пакеты разных типов и позволяет добавлять, искать, удалять, выбирать и анализировать эти пакеты.

Основной язык: C++20. Система сборки: CMake. Интерфейс: консольная диалоговая программа.

## Ограничения задания

В работе реализована основная часть лабораторной:

- UML-проектирование;
- классы логики;
- собственный шаблонный контейнер;
- диалоговая прикладная программа;
- тестирование;
- документация;
- многопоточность.

Дополнительные задания «Динамическая загрузка» и «Плагин» не выполнялись.

## Предметная область

Сервер имеет:

- сетевое имя;
- сетевой адрес;
- таблицу передач.

Таблица передач содержит элементы с указателями на описатели пакетов. Поддерживаются три типа пакетов:

- пакет почты;
- пакет файла;
- пакет гипертекста.

Пакет почты содержит адреса отправителя и получателя, имя пользователя и сообщение. Пакет файла содержит адреса, тип кода, тип информации и сообщение. Пакет гипертекста дополнительно содержит список ссылок.

## Архитектура

Проект разделен по схеме MVC:

- `model` - предметная область и бизнес-логика;
- `view` - консольный ввод/вывод;
- `controller` - обработка команд пользователя.

Такое разделение позволяет тестировать классы логики отдельно от пользовательского интерфейса.

## Основные классы

### `Packet`

Абстрактный базовый класс для всех пакетов. Хранит:

- адрес отправителя;
- адрес получателя;
- описатель сообщения.

Основные методы:

- `type()`;
- `describe()`;
- `senderAddress()`;
- `recipientAddress()`;
- `message()`.

### `MailPacket`

Наследник `Packet` для пакета почты. Дополнительно хранит имя пользователя длиной до 20 символов.

Особенность: умеет преобразовываться в `FilePacket`.

### `FilePacket`

Наследник `Packet` для файлового пакета. Дополнительно хранит:

- тип кода: `ASCII` или `BIN`;
- тип информации: управляющая или данные.

Особенность: умеет преобразовываться в `HypertextPacket`.

### `HypertextPacket`

Наследник `Packet` для гипертекстового пакета. Дополнительно хранит:

- тип кода;
- тип информации;
- список описателей ссылок.

### `MessageDescriptor`

Описатель сообщения. Хранит текст сообщения и возвращает его длину.

### `LinkDescriptor`

Описатель ссылки. Хранит протокол `FTP` или `HTTP` и сетевое имя сервера ссылки.

### `TransmissionTable`

Таблица передач. Хранит пакеты по ключу:

- адрес получателя;
- тип пакета.

Поддерживает:

- вставку;
- поиск;
- удаление;
- обход через итератор;
- текстовое описание содержимого.

### `ServerDescriptor`

Полное описание сервера. Хранит имя, адрес и таблицу передач. Выполняет операции уровня приложения:

- добавить пакет;
- найти пакет;
- удалить пакет;
- выбрать пакет по адресу получателя с приоритетом `hypertext`, `file`, `mail`;
- вывести отправителей;
- посчитать процентное соотношение пакетов.

### `PacketStatistics`

Сервисный класс для расчета статистики по таблице передач. Есть две реализации:

- последовательная;
- многопоточная.

## Собственный шаблонный контейнер

Контейнер `HashTable<Key, Value>` находится в `include/network_server/container/HashTable.hpp`.

Свойства контейнера:

- шаблонный класс;
- внутренне не использует STL-контейнеры;
- бакеты хранятся в динамическом массиве;
- коллизии разрешаются цепочками;
- хеширование выполняется аддитивным перемешиванием;
- поддержан `forward_iterator`.

Категория `forward_iterator` выбрана потому, что таблица обходится последовательно по бакетам и цепочкам. Случайный доступ для такой структуры неестественен.

## Диалоговая программа

Исполняемый файл: `network_server_dialog`.

Меню поддерживает:

- добавление пакета;
- поиск пакета по адресу получателя и типу;
- удаление пакета;
- выбор пакета по приоритету;
- преобразование пакетов;
- вывод таблицы передач;
- вывод отправителей;
- расчет процентного состава пакетов;
- выход.

При закрытом входном потоке программа завершается штатно. Это нужно для автоматических проверок и запуска через скрипты.

### Что вводит пользователь

Программа работает через числовое меню. После запуска пользователь видит:

```text
Network server dialog
1. Add packet
2. Find packet by recipient and type
3. Delete packet by recipient and type
4. Select packet by recipient priority
5. Convert packet
6. Show transmission table
7. Show senders
8. Show packet type percentages
0. Exit
>
```

Основные вводимые значения:

- `0` - завершить программу;
- `1` - добавить пакет;
- `2` - найти пакет;
- `3` - удалить пакет;
- `4` - выбрать и удалить пакет по приоритету `hypertext`, `file`, `mail`;
- `5` - преобразовать пакет;
- `6` - показать таблицу;
- `7` - показать отправителей;
- `8` - показать проценты по типам пакетов.

При выборе типа пакета используется меню:

- `1` - `mail`;
- `2` - `file`;
- `3` - `hypertext`.

При выборе типа кода:

- `1` - `ASCII`;
- `2` - `BIN`.

При выборе типа информации:

- `1` - управляющая информация;
- `2` - данные.

При выборе протокола ссылки:

- `1` - `FTP`;
- `2` - `HTTP`.

Адреса отправителя и получателя вводятся как непустые строки. В учебной реализации они не валидируются как строгие IPv4-адреса, поэтому для проверки можно вводить строки вида `192.168.1.10` и `10.0.0.5`. Имя пользователя в `MailPacket` ограничено 20 символами.

### Ожидаемый результат ручной проверки

Минимальная проверка:

```text
1
1
192.168.1.10
10.0.0.5
Hello mail
dashima
6
8
0
```

Ожидаемое поведение:

- после добавления пакета выводится `Packet added.`;
- пункт `6` показывает таблицу с одним пакетом `type: mail`;
- в таблице видны `sender: 192.168.1.10`, `recipient: 10.0.0.5`, `message: Hello mail`, `user name: dashima`;
- пункт `8` показывает `Total packets: 1`, `mail: 100%`, `file: 0%`, `hypertext: 0%`;
- пункт `0` завершает программу сообщением `Done.`.

Проверка приоритетного выбора:

1. Добавить несколько пакетов с одним адресом получателя, но разными типами.
2. Выбрать пункт `4`.
3. Ввести этот адрес получателя.
4. Программа должна выбрать пакет в порядке приоритета: сначала `hypertext`, затем `file`, затем `mail`.

Проверка преобразования:

- для `mail` пакет можно преобразовать в `file`;
- для `file` пакет можно преобразовать в `hypertext`;
- при успешном преобразовании старый пакет удаляется из таблицы, а новый добавляется.

## Многопоточность

Многопоточная операция реализована в `PacketStatistics::calculateParallel`.

Логика:

- один поток считает пакеты почты;
- второй поток считает пакеты файла;
- третий поток считает пакеты гипертекста.

Потоки только читают таблицу через `const_iterator`, поэтому состояние таблицы не изменяется и состязаний данных нет.

## Тестирование

Тесты находятся в `tests`.

Покрываются:

- вставка, поиск, замена, удаление и обход `HashTable`;
- валидация пакета почты;
- преобразование `MailPacket -> FilePacket`;
- преобразование `FilePacket -> HypertextPacket`;
- приоритетный выбор пакета сервером;
- совпадение последовательной и многопоточной статистики.

Команда запуска:

```bash
ctest --test-dir build --output-on-failure
```

## Скрипт запуска через WSL

Файл `run-wsl.ps1` предназначен для запуска проекта из Windows PowerShell.

Пример:

```powershell
.\run-wsl.ps1
```

Для проверки без запуска диалога:

```powershell
.\run-wsl.ps1 -RunTests -NoRun
```

Скрипт умеет:

- проверить WSL;
- установить Ubuntu-дистрибутив, если он отсутствует;
- поставить необходимые пакеты в Ubuntu;
- собрать проект;
- запустить тесты;
- запустить диалоговую программу.
