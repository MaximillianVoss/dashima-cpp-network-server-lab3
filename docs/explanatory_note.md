# Пояснительная записка

## Тема

Лабораторная работа 3: иерархия классов. Вариант 20: сетевой сервер.

## Постановка задачи

Требуется разработать приложение, позволяющее серверу хранить и обрабатывать таблицу передач. В таблице размещаются указатели на описатели пакетов трех типов:

- пакет почты;
- пакет файла;
- пакет гипертекста.

Для прикладной программы выбран диалоговый консольный интерфейс. Дополнительные задачи "Динамическая загрузка" и "Плагин" не выполнялись по условию задания.

## Архитектура

Проект разделен по схеме MVC:

- `model` - классы предметной области: пакеты, сервер, таблица передач, статистика;
- `view` - консольное отображение и ввод;
- `controller` - обработка команд диалоговой программы.

UML-диаграмма классов находится в `docs/uml/class_diagram.puml`.

## Иерархия пакетов

Базовый абстрактный класс `Packet` хранит адрес отправителя, адрес получателя и описатель сообщения. От него наследуются:

- `MailPacket` - добавляет имя пользователя длиной до 20 символов и преобразование в `FilePacket`;
- `FilePacket` - добавляет тип кода и тип информации, поддерживает преобразование в `HypertextPacket`;
- `HypertextPacket` - добавляет тип кода, тип информации и массив описателей ссылок.

Для владения объектами пакетов используется `std::shared_ptr<Packet>`, что позволяет хранить полиморфные объекты в таблице без ручного управления памятью.

## Шаблонный контейнер

Таблица передач реализована через собственный шаблонный класс `HashTable<Key, Value>`.

Особенности:

- внутренне не используются STL-контейнеры;
- бакеты размещены в динамическом массиве;
- коллизии разрешаются цепочками;
- хеширование выполняется функцией аддитивного перемешивания;
- поддержан forward-итератор.

Категория forward iterator выбрана потому, что таблица хранит элементы в цепочках бакетов и не предоставляет естественного случайного доступа к соседним элементам.

## Операции приложения

Диалоговая программа поддерживает:

- добавление пакета в таблицу передач;
- поиск пакета по адресу получателя и типу;
- удаление пакета по адресу получателя и типу;
- выбор пакета по адресу получателя с приоритетом `hypertext`, `file`, `mail`;
- преобразование mail -> file и file -> hypertext;
- вывод всей таблицы передач;
- вывод всех отправителей;
- расчет процентного соотношения пакетов каждого типа.

## Многопоточность

Операция расчета процентного соотношения пакетов реализована в двух вариантах:

- `PacketStatistics::calculateSequential`;
- `PacketStatistics::calculateParallel`.

Параллельный вариант запускает три задачи: отдельно для почты, файлов и гипертекста. Каждая задача проходит по таблице через const-итератор и не изменяет общие данные, поэтому состязаний нет.

Результаты бенчмарка сохранены в:

- `docs/benchmark_results.csv`;
- `docs/benchmark_percentages.svg`.

## Тестирование и анализ

Unit-тесты написаны на Catch2. Проверяются:

- вставка, поиск, замена, удаление и итерация в `HashTable`;
- валидация и преобразования пакетов;
- приоритетный выбор пакета сервером;
- совпадение результатов однопоточной и многопоточной статистики.

Выполненные проверки:

- обычная сборка и `ctest`: 7/7 тестов успешно;
- AddressSanitizer/UBSan: 7/7 тестов успешно;
- ThreadSanitizer: 7 тестовых сценариев, 31 assertion успешно при запуске через `setarch $(uname -m) -R`;
- scripted-запуск диалоговой программы с командой выхода.

`clang-tidy`, `cppcheck`, `valgrind` и `doxygen` в текущем WSL-окружении не установлены. Для Doxygen подготовлен `Doxyfile`; публичные методы классов документированы в заголовочных файлах.

## Вывод

Разработано C++20-приложение для варианта "Сетевой сервер". Реализованы иерархия классов пакетов, собственная шаблонная таблица передач, диалоговый интерфейс, unit-тесты, UML-диаграмма и многопоточный расчет статистики.
